
### Структура проекта
```text
.
├── backend/                  # Всё что связанно с тем, как сервис работает внутри
│   ├── domain/service/       # Сервисы, бизнес-логика
│   ├── integration/          # Service Provider Interfaces для сервисов, абстрагируют infrastructure (postgres, ...)
│   ├── infrastructure/       # postgres, kafka, firebase и другое, с чем работают сервисы через integration слой
│   │   ├── postgres/init.sql # Инициализация постгреса, таблички, функции, триггеры, ...
├── common/types              # Переиспользуемые модельки, в основном не связанные с доменом, переиспользуются на бэке и фронте
├── containers/               # application-модули конечных приложений-частей бэка, например отдельных docker контейнеров.
│                             # Пока что тут только monolith - он создаёт соединение со всей инфраструктурой,
│                             # запускает сервисы, запускает gateway (rpc api).
├── presentation/             # Всякие штуки, с помощью чего фронт общается с сервисом. DTO, Rpc-интерфейсы.
├── sdk/                      # SDK, его использует фронт. Класс Client.
├── libs/                     # Библиотеки не связанные с доменом. Переиспользуется на фронте. Например Optional<T> класс.
└── play/                     # Playground для тестов
```
